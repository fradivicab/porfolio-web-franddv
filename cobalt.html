Proyecto: Cobalt

Rol: QA API Analist

Cobalt es una empresa de ciberseguridad especializada en pentesting moderno bajo un modelo llamado Pentesting as a Service (PtaaS).

En vez de pruebas puntuales una vez al año, Cobalt propone pentesting continuo, integrado al ciclo de desarrollo.

El trabajo analisis API permitio: validar integracion entre en Jira y sistemas internos de cobalt, haciendo asi la informacion mas robusta en informes, graficos y datos empresariales.

Mike Garon

TL - Cobalt Proyect

Rol de QA API dentro del proyecto

En el proyecto me desempeñé como QA API Analyst, siendo responsable de validar la correcta integración entre múltiples servicios backend a través de pruebas exhaustivas de APIs. El trabajo se centró en asegurar la calidad, consistencia y confiabilidad de los endpoints desarrollados, los cuales soportan flujos críticos del negocio dentro de la plataforma.

La validación se realizó principalmente mediante pruebas manuales con Postman, analizando requests, responses, estados HTTP y estructuras de datos, con foco en detectar fallos de integración, errores lógicos y comportamientos inesperados entre servicios.

  • Ejecución de pruebas manuales de APIs REST utilizando Postman para validar servicios de integración desarrollados internamente.
  • Verificación de flujos completos de negocio a nivel backend, asegurando la correcta comunicación entre distintos servicios y sistemas.
  • Análisis y validación de respuestas JSON, estructuras de datos, campos obligatorios y reglas de negocio.
  • Pruebas de distintos escenarios funcionales, incluyendo casos positivos, negativos y edge cases.
  • Validación de autenticación, autorización y manejo de headers, tokens y parámetros dinámicos.
  • Detección, documentación y seguimiento de defectos mediante JIRA, colaborando activamente con los equipos de desarrollo.
  • Re-ejecución de pruebas sobre servicios corregidos para validar fixes y evitar regresiones.
  • Aseguramiento de la consistencia de los servicios de integración frente a distintos estados del sistema.
  • Colaboración cercana con desarrolladores y stakeholders técnicos para comprender el comportamiento esperado de los endpoints y ajustar los criterios de aceptación.

Enfoque de calidad

Dado que se trata de una plataforma orientada a seguridad y servicios críticos, la precisión en las respuestas de las APIs, el manejo correcto de estados y la estabilidad de las integraciones resultaron fundamentales para garantizar un funcionamiento confiable del producto y una correcta experiencia para los usuarios finales y sistemas consumidores.
