Proyecto: DengueChat – Mobile & Web Applicationb

Rol: QA Engineer

DengueChat+ es una aplicación mobile (Android & iOS) y web orientada a la prevención, seguimiento y comunicación relacionada con el dengue, desarrollada en colaboración con Tulane University (USA). El producto combina funcionalidades de interacción con usuarios finales y un portal web de administración, permitiendo la gestión de datos, monitoreo de información y soporte a campañas de salud pública.

DengueChat usa elementos de juegos y redes sociales para aumentar la motivación de los participantes en tres niveles de involucramiento, el de la vigilancia comunitaria, a nivel de casas e individuales. DengueChat reconoce y recompensa las acciones específicas de intervención contra el vector y otorga puntos, medallas, títulos, y otras distinciones en el sitio de internet.

Rol de QA dentro del proyecto

Como QA Engineer, fui responsable de asegurar la calidad funcional, visual y técnica del producto en todas sus plataformas, trabajando de forma transversal entre mobile, web y backend.

Testing funcional y manual

  • Creación, diseño y ejecución de casos de prueba manuales para:
    • Aplicación móvil Android
    • Aplicación móvil iOS
    • Plataforma web
  • Validación de flujos completos de usuario, asegurando coherencia funcional entre las distintas plataformas.
  • Validación de respuestas del backend ante acciones del usuario, asegurando coherencia de datos y manejo correcto de errores.
  • Ejecución de pruebas de regresión ante cada nueva versión del producto.

Testing de APIs y backend

  • Pruebas de APIs y servicios utilizando Postman, validando:
    • Correcta comunicación entre frontend y backend
    • Estructura y contenido de las respuestas
    • Manejo de errores y estados inesperados
  • Uso de Sentry para:
    • Análisis de logs del backend
    • Identificación de excepciones y errores en tiempo real
    • Trazabilidad de errores desde el frontend hasta los servicios backend desplegados en AWS

Debugging y análisis de errores

  • Debugging de la aplicación Android mediante Android Studio, analizando:
    • Logs
    • Comportamiento de la app en ejecución
    • Manejo de errores y estados inesperados
  • Investigación de errores reportados por usuarios o detectados durante las pruebas, identificando la causa raíz y colaborando con el equipo de desarrollo para su resolución.

Testing mobile & distribución de builds

  • Obtención de builds móviles mediante Expo, generando:
    • APK para Android
    • Builds para iOS
  • Uso de TestFlight para:
    • Distribución y prueba de versiones iOS en dispositivos reales
    • Validación del comportamiento en distintos modelos y versiones del sistema operativo

Testing web y revisión UX/UI

  • Validación funcional y visual de la versión web de administración.
  • Revisión detallada de:
    • Aspectos gráficos
    • Consistencia visual
    • Experiencia de usuario (UX)
  • Propuesta de mejoras no solo a nivel de interfaz, sino también:
    • Cambios estructurales en el código
    • Ajustes para mejorar mantenibilidad y comportamiento del frontend

Metodología y enfoque

El enfoque de testing se basó en:

  • Cobertura completa del producto en mobile, web y backend
  • Análisis técnico profundo de errores, más allá del síntoma visible
  • Comunicación activa con el equipo de desarrollo
  • Mentalidad de calidad como parte del producto, no como etapa final

Tecnologías y prácticas

  • Testing: Manual Testing, Frontend Testing, Backend Testing, API Testing
  • Mobile: Android Studio, Expo, TestFlight
  • APIs: Postman
  • Observabilidad: Sentry (frontend & backend logs)
  • Cloud: AWS
  • Análisis: Log Analysis, Bug / Issue Tracking
